# AI SEO Public Deployment

## Production Topology

Ansiversa public AI knowledge is generated by the backend from the Canonical AI
Knowledge Registry. The canonical public URLs are on the web domain:

```text
https://ansiversa.com/robots.txt
https://ansiversa.com/llms.txt
https://ansiversa.com/llms-full.txt
https://ansiversa.com/ai-sitemap.xml
https://ansiversa.com/public-ai-knowledge.json
https://ansiversa.com/public-ai-jsonld.json
https://ansiversa.com/public-ai-metadata.json
```

The backend also serves the same registry-derived content at:

```text
https://api.ansiversa.com/robots.txt
https://api.ansiversa.com/llms.txt
https://api.ansiversa.com/llms-full.txt
https://api.ansiversa.com/ai-sitemap.xml
https://api.ansiversa.com/public-ai-knowledge.json
https://api.ansiversa.com/public-ai-jsonld.json
https://api.ansiversa.com/public-ai-metadata.json
https://api.ansiversa.com/api/v1/knowledge/public
https://api.ansiversa.com/api/v1/knowledge/public/jsonld
https://api.ansiversa.com/api/v1/knowledge/public/metadata
```

The frontend Vercel deployment rewrites only the approved AI public paths to the
backend. The normal React SPA fallback remains responsible for application
routes.

## Artifact Contract

Expected content types:

```text
robots.txt                       text/plain
llms.txt                         text/plain
llms-full.txt                    text/plain
ai-sitemap.xml                   application/xml
public-ai-knowledge.json         application/json
public-ai-jsonld.json            application/ld+json
public-ai-metadata.json          application/json
/api/v1/knowledge/public         application/json
```

Generated artifacts must publish exactly 100 public apps and 14 public
categories. Bill Splitter must be present. Public wording must describe the
fixed catalog as exactly 100 solution apps, never `100+`.

## Cache Policy

Public AI artifacts are safe for public caching with bounded freshness:

```text
Cache-Control: public, max-age=3600
X-Robots-Tag: index, follow
```

Artifacts are not content-hashed, so they must not use immutable or year-long
caching. Authenticated Assistant and user APIs must not inherit this public
artifact cache policy.

## Robots And Sitemap

`robots.txt` allows public crawling and references:

```text
Sitemap: https://ansiversa.com/ai-sitemap.xml
LLMs: https://ansiversa.com/llms.txt
```

`ai-sitemap.xml` contains the home page, About, Pricing, FAQ, Contact, Apps,
and every public app overview URL. It uses only canonical HTTPS production URLs
on `ansiversa.com`.

## Verification Command

Run the read-only smoke verifier after deployment:

```bash
python -m app.modules.knowledge.verify_public_deployment \
  --base-url https://ansiversa.com \
  --api-base-url https://api.ansiversa.com
```

The verifier fetches approved public artifacts, follows at most five redirects,
checks status and content type, rejects SPA HTML fallback, validates JSON/XML
and JSON-LD structure, confirms the 100-app and 14-category counts, confirms
Bill Splitter, checks canonical HTTPS URLs, and scans for forbidden public
content patterns.

Crawler user-agent spot checks can use the same command with
`--user-agent`. This is only a hosting accessibility check and is not a crawler
bypass mechanism.

## Search Console Checklist

Manual steps for the Partner:

1. Verify the `ansiversa.com` domain property.
2. Inspect `https://ansiversa.com/robots.txt`,
   `https://ansiversa.com/llms.txt`, and representative app pages.
3. Submit or resubmit the standard sitemap.
4. Submit `https://ansiversa.com/ai-sitemap.xml` only if Search Console accepts
   it as useful.
5. Request indexing for Home, About, Apps, Bill Splitter, and representative
   app overview pages.
6. Monitor crawl errors and canonical selection.
7. Compare indexed wording against the fixed exactly-100 catalog language.

`llms.txt` is a public crawler artifact, not a Search Console submission type.

## Monitoring Cadence

Recommended checks:

```text
immediately after deployment
7 days after deployment
30 days after deployment
after major public knowledge changes
```

Track indexed page count, stale wording, app recognition, incorrect claims,
citation sources, canonical mismatches, crawler errors, sitemap status, and
whether public artifacts still match the committed generated output.

## Known Limitations

Search engines and AI answer systems may take days or weeks to crawl and
refresh. Passing the smoke verifier proves public reachability and artifact
validity; it does not guarantee indexing, ranking, rich-result eligibility, or
answer adoption.
